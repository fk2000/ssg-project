# Markdown と SSG の相性

SSG を使う上で、コンテンツの記述言語として Markdown は標準的な選択肢となっています。なぜこれほどまでに支持されているのか、その理由を紐解きます。

## 1. Markdown の特性

- **シンプル設計**: 記号を少し覚えるだけで、構造化された文章が書けます。
- **書きやすさ**: HTML タグをいちいち閉じる手間がなく、思考を妨げずに執筆に集中できます。
- **Git との親和性**: プレーンテキストであるため、Git での差分（diff）が非常に見やすく、変更履歴の管理が容易です。

:::message
Markdown は技術文書だけでなく、個人のメモやドキュメントなど、あらゆるテキスト管理に役立ちます。一度覚えれば一生モノのスキルになります。
:::

## 2. なぜ SSG と相性が良いのか？

SSG の最大の役割は、**「コンテンツ（Markdown）」と「見た目（テンプレート）」を分離すること**にあります。

1. **爆速のビルド**: シンプルなテキスト解析で済むため、大量のページも高速に HTML 化できます。
2. **デザインの統一**: 全ページ同じ CSS/コンポーネントが適用されるため、Markdown 側では内容に集中できます。
3. **SEO への強さ**: Markdown の構造がそのまま正しいセマンティックな HTML (h1, h2等) に変換されるため、自然と SEO に強いサイトになります。

:::message alert
ただし、画像パスの指定方法などは SSG ツールによって微妙に異なる場合があるため、各ツールのドキュメントを確認することが重要です。
:::

## まとめ

Markdown で書き、SSG で配信する。この組み合わせは、現代の技術者にとって最も合理的で強力な「武器」の一つと言えるでしょう。
