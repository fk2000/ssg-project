# Web の仕組みを理解する

SSG に取り掛かる前に、私たちが普段使っている Web がどのような仕組みで動いているのか、最低限知っておくべき知識を整理します。

## HTTP とサーバーの役割

ブラウザがページを表示するとき、裏側では「リクエスト（お願い）」と「レスポンス（返事）」というやり取りが行われています。

- **リクエスト**: ブラウザがサーバーに「このページを見せて」と頼む。
- **レスポンス**: サーバーがブラウザに「はい、これが HTML です」と返す。

## 静的サイトと動的サイトの違い

- **静的サイト**: 誰がいつアクセスしても同じ HTML ファイルを返す。SSG はこれを作ります。
- **動的サイト**: アクセスするたびにサーバーがその場で HTML を組み立てる（WordPressなど）。

:::message
静的サイトは「作り置きの料理」のようなものです。注文されてから作る必要がないため、非常に素早く提供できます。
:::
