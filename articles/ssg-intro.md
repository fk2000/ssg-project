# 静的サイトゲネレーター（SSG）とは？

現代の Web 制作において、WordPress のような動的 CMS に代わって注目されているのが **SSG (Static Site Generator)** です。

## なぜ今、SSG なのか？

1. **爆速の表示スピード**: あらかじめ HTML を生成しておくため、サーバーでの計算待ちがありません。
2. **高いセキュリティ**: データベース（DB）がないため、SQL インジェクションなどの脆弱性を根本から排除できます。
3. **低コスト**: 物理的なサーバーが不要で、Cloudflare Pages や Vercel などのホスティングサービスで安価に（あるいは無料で）公開できます。

## どのように動作するのか？

- **開発時**: Markdown で原稿を書き、テンプレート（CSSやJS）と組み合わせます。
- **ビルド時**: SSG ツールがそれらを解析し、全ページの HTML を出力します。
- **公開時**: 生成された HTML ファイル群をサーバー（CDN）に配置します。

:::message
仕組みは非常にシンプルですが、そのシンプルさゆえに、メンテナンス性が高く、長期にわたって安定した運用が可能です。
:::
